# Phoenix AI v1.0.0-beta

Status: release candidate

## Objetivo

Preparar a Phoenix AI para implantacao controlada com API, Studio e Worker executando em ambientes reproduziveis.

## Entregas

- Dockerfiles para API, Studio e Worker.
- `docker-compose.yml` para desenvolvimento operacional.
- `docker-compose.prod.yml` para implantacao com volumes nomeados.
- Validacao centralizada de variaveis de ambiente.
- Endpoint publico `GET /version`.
- Endpoint operacional `GET /health/details`.
- Scripts de backup, restore, integridade e diagnostico.
- Tela `Studio > Sistema`.
- Headers HTTP de seguranca.
- Rate limiting configuravel.
- Pipeline de qualidade expandido com integridade, diagnostico e build Docker.

## Comandos

```bash
pnpm run studio:build
pnpm run integrity:check
pnpm run diagnostics
pnpm run backup
docker compose up --build
```

## Observacoes

- O restore exige confirmacao explicita com `--yes` ou `PHOENIX_RESTORE_CONFIRM=true`.
- `PHOENIX_API_KEY` e obrigatoria em `PHOENIX_ENV=production`.
- Os volumes `.storage`, `reports` e `output` devem ser persistidos fora do container.
